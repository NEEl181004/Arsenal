TEXT: 1. Tool Overview | IMGS: []
TEXT: Description | IMGS: []
TEXT: Auditd (Audit Daemon) is a Linux-based auditing tool used to monitor and log system activity. It is a key component of the Linux Auditing System and helps security professionals, system administrators, and forensic analysts track events such as unauthorized access, policy violations, and system modifications. Auditd provides real-time monitoring of files, users, and processes to ensure compliance with security policies and regulatory requirements. | IMGS: []
TEXT: Auditd can be configured to log specific activities, generate alerts for suspicious behavior, and integrate with SIEM (Security Information and Event Management) solutions. Its flexibility and detailed logging capabilities make it an essential tool for system hardening and incident response. | IMGS: []

TEXT: Initial Setup | IMGS: []
TEXT: Auditd runs as a background service and starts automatically upon installation. It uses rule-based configuration to define which activities should be monitored. | IMGS: []
TEXT: Configuration Files | IMGS: []
TEXT: Main Configuration: /etc/audit/auditd.conf | IMGS: []
TEXT: Rules Configuration: /etc/audit/rules.d/audit.rules | IMGS: []
TEXT: Common Configuration Options | IMGS: []

TEXT: max_log_file = 10 | IMGS: []
TEXT: max_log_file_action = ROTATE | IMGS: []

TEXT: Defining Audit Rules | IMGS: []
TEXT: To monitor a specific file or directory: | IMGS: []
TEXT: auditctl -w /etc/passwd -p wa -k passwd_changes | IMGS: []
TEXT: -w /etc/passwd → Watch file | IMGS: []
TEXT: -p wa → Monitor write and attribute changes | IMGS: []
TEXT: -k passwd_changes → Tag rule with a key | IMGS: []
TEXT: To make rules persistent, add them to /etc/audit/rules.d/audit.rules. | IMGS: []
